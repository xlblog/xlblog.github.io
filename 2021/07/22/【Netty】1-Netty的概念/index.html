【Netty】1.Netty的概念

发表于 2021-07-22 更新于 2021-07-26 分类于 Netty

什么是Netty

Netty是一款异步的事件驱动的网络应用程序框架，支持快速地开发可维护的、高性能的、面向协议的服务器和客户端。

Netty是基于NIO实现的一个应用框架。

Java NIO

早期的网络编程，是一个阻塞式的编程方式。即：java.net包下的Socket编程。

它的连接方式如下图：

这种阻塞式的I/O的缺点：

  1. 在任何时候都可能有大量的线程处于休眠状态，只是等脑袋输入或者输出数据就绪，比较浪费资源
  2. 需要为每个线程的调用栈都分配内存
  3. 即使Java虚拟机在物理上可以支持非常大数量的线程，但远在达到该极限之前，上下文切换所带来的开销就会有很多问题

Java的非阻塞I/O是在jdk1.4引入的，所在的包是：java.nio。

非阻塞设计如下图：

Selector是java的非阻塞I/O实现的关键。它使用了事件通知API以确定在一组非阻塞套接字中有哪些已经就绪能用进行I/O相关的操作。因为可以在任何的时间检查任意的读写操作的状态，所以，一个单一的线程便可以处理多个并发的连接。

非阻塞I/O的优点：

  1. 使用较少的线程便可以处理许多连接，因此也减少了内存管理和上下文切换带来的开销
  2. 当没有I/O操作需要处理的时候，线程也可以被用于其他任务

虽然有很多基于NIO的应用程序，但是想要快速做到正确和安全并不容易，特别是，在高负载下可靠和高效的处理和调度I/O操作是一项繁琐而且容易出错的任务，因此，像这种网络编程，最好是使用Netty来完成。

Netty的核心组件

Channel

Channel是Java NIO的一个基本构造，它代表一个到实体(如一个硬件设备、一个文件、一个网络套接字)的开放连接，如读操作和写操作。可以把Channel看做是传入或者传出数据的载体。因此它可以被打开或者关闭，可以被连接或者断开连接。

Callback

callback是一个简单的方法，提供给另一种方法作为引用，这样后者就可以在某个时间调用前者。

Netty内部使用了回调来处理事件：当一个回调被触发时，相关的事件可以被一个InterfaceChannelHandler的实现处理。例如，一个新的连接建立时，ChannelHandler的channelActive()回调方法将会被调用。

Future

Future是另一种在操作完成时通知应用程序的方式。Netty的实现：ChannelFuture，用于在执行异步操作的时候使用。

Netty中的ChannelFuture提供了多个方法，我们可以在它上面注册一个或者多个ChannelFutureListener实例。它和Callback是一种互相补充的作用。

Event和Handler

Netty使用不同的事件来通知我们更改的状态或操作的状态。我们可以通过这些状态的变化，来进行对应的操作。

Netty的ChannelHandler是各种处理程序的基本抽象。
